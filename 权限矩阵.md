# IvyEuro 网站与知识引擎权限矩阵

## 目标

- 默认只能看自己的内容
- 管理员可以看全部内容
- 所有需要后台管理的内容都纳入同一套权限体系
- 公开内容与内部内容分层管理

## 角色定义

| 角色 | 说明 | 可见范围 | 可操作范围 |
|---|---|---|---|
| `viewer` | 只读用户 | 仅看公开内容，或被授权内容 | 不能编辑 |
| `editor` | 内容编辑 | 看自己负责的内容 + 公开内容 | 新增 / 编辑自己负责内容 |
| `admin` | 管理员 | 看全部内容 | 新增 / 编辑 / 删除全部内容，分配权限 |
| `super_admin` | 系统管理员 | 看全部内容 + 系统配置 | 管理账号、角色、系统配置、审计 |

## 内容分类

| 内容类型 | 示例 | 默认可见 | 默认可编辑 |
|---|---|---|---|
| 公开内容 | 首页、课程介绍、新闻动态、老师公开页、联系页 | 所有人 | `editor` / `admin` |
| 内部内容 | 导入文件、分析报告、历史记录、草稿 | 对应 owner + `admin` | owner / `admin` |
| 敏感内容 | 微信聊天原文、手机号、学员资料、导出文件 | owner + `admin` | owner / `admin` |
| 系统内容 | 账号、角色、权限、审计日志、配置项 | `admin` / `super_admin` | `super_admin` |

## 权限矩阵

### 内容查看权限

| 内容类型 \ 角色 | viewer | editor | admin | super_admin |
|---|---:|---:|---:|---:|
| 公开内容 | 可看 | 可看 | 可看 | 可看 |
| 内部内容 | 不可看 | 仅自己 | 全部可看 | 全部可看 |
| 敏感内容 | 不可看 | 仅自己 | 全部可看 | 全部可看 |
| 系统内容 | 不可看 | 不可看 | 可看 | 可看 |

### 内容编辑权限

| 内容类型 \ 角色 | viewer | editor | admin | super_admin |
|---|---:|---:|---:|---:|
| 公开内容 | 不可改 | 仅自己负责内容 | 全部可改 | 全部可改 |
| 内部内容 | 不可改 | 仅自己 | 全部可改 | 全部可改 |
| 敏感内容 | 不可改 | 仅自己 | 全部可改 | 全部可改 |
| 系统内容 | 不可改 | 不可改 | 限制编辑 | 全部可改 |

## 对网站各模块的建议权限

| 模块 | 权限建议 |
|---|---|
| 首页 / 公共页面 | 所有人可见，少数编辑可改 |
| 服务项目 / 课程介绍 | 编辑可改，管理员审核发布 |
| 新闻动态 / 文章 | 编辑可写，管理员可审 |
| 老师简介 | 编辑可写，管理员可审 |
| 课程表 | 编辑可写，管理员可审，过期自动归档 |
| 家长好评 | 编辑可写，管理员可审 |
| 联系方式 | 管理员可改，避免误改联系方式 |
| 表单 / 预约 | 仅管理员和授权编辑可看后台数据 |
| 微信聊天记录 | 仅 owner + 管理员可看 |
| 分析报告 | 仅 owner + 管理员可看 |
| 历史导入记录 | 仅 owner + 管理员可看 |

## 后台结构建议

```text
登录
  └─ 角色判断
       ├─ viewer
       │    └─ 仅可访问公开页和只读面板
       ├─ editor
       │    └─ 只看自己负责的内容与导入记录
       ├─ admin
       │    ├─ 全部内容管理
       │    ├─ 全部导入记录
       │    └─ 权限管理
       └─ super_admin
            ├─ 账号管理
            ├─ 权限管理
            ├─ 系统配置
            └─ 审计日志
```

## 数据字段建议

每条内容或导入记录至少保存：

- `id`
- `owner_id`
- `owner_name`
- `content_type`
- `visibility`
- `created_at`
- `updated_at`
- `status`
- `tags`
- `source_file`
- `audit_log_id`

## 落地顺序

1. 先加登录
2. 再加 `owner_id`
3. 再加 `viewer/editor/admin/super_admin`
4. 再区分公开、内部、敏感内容
5. 最后加审计日志和导出限制

## 结论

- 网站所有内容都应该纳入同一套权限体系
- 公开内容、内部内容、敏感内容要分层
- 普通用户只能看自己的
- 管理员可以看全部
- 系统管理员负责权限和配置
